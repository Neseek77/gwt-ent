Messages
  Method override in GWT message class? Not tested

  In GWT implement, framework supposed every template message should exists
  in a GWT message class as a Method.
  The method not support "constraint.xxxx", I changed the message id 
  from "constraint.xxxx" to "constraint_xxxx"
  
  constraint_min=must be greater than or equal to {value}
  Changed to:
  constraint_min=must be greater than or equal to {0}
  
  
Using Messages
  If you just want using the messages provided by ThirdParty(GWTENT or Other Extend)
  please using 
    GWTValidateMessageStore.addMessageObject(YouMessageInstance);
    GWTValidateMessageStore.addMessageObject(GWT.create(ValidateMessages.class));
    GWTValidateMessageStore.addMessageObject(GWT.create(OtherThirdPartyMessaegs.class));
    GWTValidateMessageStore.addMessageObject(GWT.create(OtherThirdPartyMessaegs1.class));
    
  Please Note, you can provide a function with the same id to override ThirdParty' message
  
  
  If you want control all messages or don't want validate message store in so many files,
  you can create your message class which contain all messages. then:
  GWTValidateMessageStore.addMessageObject(YouMessageInstance);
